# 按「音画同步完整设计方案.txt」实施

## 核心原则（来自方案）
- 音频恒速播放，视频追着音频走
- 4 层架构：Layer 0 音频设备 → Layer 1 同步决策 → Layer 2 硬件时钟 → Layer 3 精确等待

## 实施步骤

### Step 1: 创建 sync 模块目录和文件
```
src/sync/
├── audio_clock.h / .c     时钟抽象（自动降级）
├── hw_clock.h / .c        WASAPI 硬件时钟（复用 ao_wasapi.c 的逻辑）
├── wo_clock.h / .c        waveOut 软件时钟（fallback）
├── sync.h / .c            同步决策引擎
├── precise_sleep.h / .c   高精度等待
└── sync_debug.h / .c      调试日志（可选）
```

### Step 2: 实现 hw_clock.c
复用 ao_wasapi.c 中的 WASAPI 逻辑，但加入关键改进：
- `hwclock_write()` 每次写入更新 `pts_base` + `device_base`
- `hwclock_get_time()` = `pts_base + (device_time - device_base) - hw_delay`
- 这比当前的 `last_write_pts - ring_buffered` 更精确

### Step 3: 实现 sync.c
`sync_video()` 返回 SyncDecision (RENDER/WAIT/DROP)，基于同步窗口参数。

### Step 4: 实现 precise_sleep.c
QPC + Sleep + spin，精度 ±0.5ms。

### Step 5: 修改 player.c
- 用 SyncContext 替代当前的 ad-hoc 同步逻辑
- 用 sync_video() 替代 video_timer_cb 中的 diff 判断
- 用 precise_sleep 替代 Sleep

### Step 6: 修改 ao_wasapi.c
- 将 ring buffer 写入改为通过 hw_clock 或等效的 PTS-based 时钟
- 或保持当前 ring buffer 方案但改进 ao_get_clock()

### Step 7: 更新 build.bat
添加 sync 模块源文件和 winmm.lib

## 关键文件
- `src/sync/` (新建 8 个文件)
- `src/core/player.c` (修改同步逻辑)
- `src/audio_out/ao_wasapi.c` (改进时钟)
- `build.bat` (添加源文件)
